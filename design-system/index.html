1. Foundations

Core design tokens. These compile from SCSS variables in src/styles/_variables.scss and are the single source of truth for all visual decisions.

Colour palette

Colours are defined as SCSS variables. The primary indigo #2c2d84 is used for structural backgrounds (header, footer, banners) and as the default link colour. The hover blue #006ab4 is strictly a hover/interactive state — never use as a static colour. Yellow #ffdd00 is reserved for focus rings only.

Brand

$color-primary #2c2d84 Links, icons, structural bg
$color-primary-dark #006ab4 Hover states only
$color-primary-light #bfdaec Tinted backgrounds
$color-midnight-blue #1d1751 Ask button, chat panel

Accent & action

$color-border #b1b4b6 Borders, dividers
$color-border-light #e8e8e8 Subtle dividers, card borders

Typography

The root font size is set to 112.5% (18px). All rem values scale from this. The type system uses a single sans-serif stack — no web fonts are loaded, keeping performance lean for the prototype. Font weight 800 is used for all heading levels.

Body text — used for all paragraph content. Line height 1.5. Maximum line width is $max-width-narrow (768px) for prose columns.

Small text — metadata, descriptions, footer links, badge labels.

link

Default link style — indigo #2c2d84, underline at 1px. Hover turns blue #006ab4 with underline at 2px.

Breakpoints

Mobile-first. Use @include bp(md) for most layout switches. The lg breakpoint is used for multi-column grids. All breakpoints use min-width.

$bp-sm480px — small phones in landscape
$bp-md768px — tablet / most layout switches (nav → desktop, 2-col grids)
$bp-lg1024px — desktop (3-col grids, sticky sidebar, wider hero)
$bp-xl1280px — wide desktop (rarely needed)
$max-width1090px — maximum content width, applied via .container

2. Components

Reusable UI components. Each has a base class and optional modifier classes. Context overrides (e.g. dark-background reversals) are applied via ancestor selectors.

Breadcrumb

Two visual variants sharing the same markup — the colour flips based on the modifier class. Use aria-current="page" on the last item when the current page is shown. The final crumb is typically omitted to keep the trail concise.

.breadcrumb (on dark page banner)

  1. Home
  2. Waste and recycling

.breadcrumb--on-light (on white content header)

  1. Home
  2. Waste and recycling
  3. Bin collections

Alert

Used to communicate time-sensitive information at the top of a page section. Always includes a dismiss button and a labelled header. Three colour variants — blue for information, amber for warnings, red for urgent/critical. Place immediately after the page header, before the breadcrumb or main content.

.alert--blue

Information notice

Enter your postcode to check your personal collection schedule and see which bins are due.

.alert--red

Bank holiday collection changes

Collections during the week of Monday 26 May will run one day late due to the bank holiday.

Primary button

Used for primary CTAs inside prose content and forms. The --accent modifier uses the lime green colour — use sparingly when you need to distinguish a positive/confirm action.

.btn-primary

Start now

.btn-primary--accent

Confirm and continue

Ask button

Opens the AI chat panel. The base style is dark midnight-blue with a green chat bubble and spark overlay icon. It reverses to a white card in all light-background contexts. The button always has data-open-chat and aria-controls="chat-panel". Never modify the icon stack — the overlaid spark is carefully positioned.

Base (dark bg — used standalone only)

.hero .ask-btn / .contact-block .ask-btn / .content-sidebar .ask-btn / .inline-ask .ask-btn (white reversal)

Report CTA

A contextual CTA for reporting problems, concerns or changes. The base style lives on dark backgrounds (page banner, section landing hero). Light-background reversals are applied via .site-nav .report-cta and .site-footer .report-cta. On hover: icon and label go to the contextual hover blue, with a lift and shadow.

Base (dark page banner background)

Report it Contact us about problems, concerns or changes

.site-nav .report-cta (navigation panel — light bg)

Report it Contact us about problems, concerns or changes

3. Tasks & services

Three related patterns for surfacing tasks, services and topics. Task tiles are for primary actions. Service rows are for browsing service areas. Topic links are for navigating child pages within a section.

Task tiles

Used in the main .top-tasks section (dark background) and on section landing page banners. Each tile has a service category label (hidden on mobile), a primary task label, and an accent-coloured icon. The grid is 2-col on mobile, 4-col on desktop. A light-background variant .top-tasks--light uses $color-bg-warm.

Waste and recycling Check bin collection day
Council tax Pay council tax
Planning View a planning application
Roads and transport Pay or appeal a parking fine

Service rows

Used in the services grid on the homepage (.popular-services--cards) and in section landing pages. Each row has an icon, name with an animated right-arrow, and an optional description. On hover the row lifts and the arrow slides right. Available modifiers: --no-icons, --compact, --cards (bordered card grid), --topics.

  • Housing and benefits Housing support and advice, benefits, and the Debt Respite Scheme.
  • Schools and learning Schools, early years and childcare, and the local offer for children with SEND.

Topic links

Used on level-2 landing pages to link to child content pages. Each card shows a name, optional description, and a right-arrow that animates on hover. Grid: 1-col on mobile, 2-col on desktop.

Check what you can put in your bins Find out what goes in each bin type — recycling, food waste, general waste and garden waste. Replace a damaged or missing bin Request a free replacement if your bin is damaged, missing or stolen.

4. Cards

Card patterns for news, featured stories, promotional content and timely service updates. Always use real images or an appropriate placeholder — never leave image slots empty.

News card

Used in the homepage news grid (.news-grid). Each card has an image, category, date, headline link, and excerpt. Grid: 1-col mobile → 2-col tablet → 3-col desktop. The .view-all-link pattern appears as a section header action link.

Highways

28 January 2025

Council highlights highways progress over the past six months

Buckinghamshire Council has shared an update on the extensive work carried out by its highways teams.

.view-all-link

All news

Featured story

Used in the .right-now section (currently commented out on homepage). A large 2-col card with a prominent image, heading, body text and CTA link. Used for seasonal or time-sensitive features.

Half term begins 16 February

School term dates, INSET days, closures, and transport information for all Buckinghamshire schools.

Check term dates

Promo banner

A horizontal promotional card — image on the left, dark content panel on the right. Used above the news section for campaign or award content. Stacks vertically on mobile.

Proud of Bucks Awards 2026

Celebrating the people and communities making a difference across Buckinghamshire.

Nominations are now open

Timely cards

Small contextual cards used in the .right-now section. A coloured left border signals urgency or category. Swap content seasonally — the card itself does not carry permanent content. Four colour variants: --blue, --green, --orange, --alert.

5. Content patterns

Patterns used on content pages and throughout the site. These are not full-page templates but reusable sections that appear across multiple page types.

Accordion

Used in content pages to structure detailed or tabular content (e.g. what goes in each bin). Uses native <details>/<summary> — no JavaScript required. The chevron rotates 90° when open via the [open] CSS selector. Group multiple items inside a single .accordion wrapper.

General waste bin

Accepts broken glass, nappies, crisp wrappers, plastic bags and similar non-recyclable household waste.

Do not put in: batteries, gas bottles, electrical items or clinical waste.

Mixed recycling bin

Accepts plastic bottles, glass bottles and jars, tins, cans, foil, aerosols, and milk cartons.

Do not put in: plastic bags, crisp packets, wrappers, or polystyrene.

Contents list

In-page navigation for long content pages. Based on the GDS pattern. Placed at the top of the .content-main column, before the first prose heading. Links should anchor to id attributes on headings within the page.

Contents

  1. General waste bin
  2. Food waste caddy
  3. Mixed recycling bin
  4. Paper and cardboard recycling
  5. Garden waste bin

Contact block & inline ask

Two patterns for surfacing the AI assistant. The .contact-block is a full-width dark footer section used as a fallback CTA at the bottom of most pages ("Can't find what you need?"). The .inline-ask is a self-contained dark card that can be dropped inside prose content or section landing pages.

.contact-block (full-width section)

Can't find what you need?

.inline-ask (embedded in content)

Not sure which bin?

Page feedback

A lightweight "Is this page useful?" strip placed immediately before the footer on all page types. Uses warm background. The Yes/No buttons can be wired to an analytics event or a follow-up form.

Is this page useful?

Related pages

A sidebar widget for content pages. Placed in .content-sidebar, which is sticky on desktop. Provides contextual navigation to related content within the same service area.
